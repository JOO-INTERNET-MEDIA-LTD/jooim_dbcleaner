# Support

The free download of this module does not include individual technical support.

Installation, configuration review, troubleshooting and custom modifications may be available as a paid service through Joobox or JOO INTERNET MEDIA LTD.

## Before requesting support

Please check the following first:

1. Confirm that the module is installed in the correct folder: `modules/jooim_dbcleaner/`.
2. Open the module configuration and save the settings once.
3. Clear the PrestaShop cache.
4. Check whether the cron token in your cron command matches the token shown in the module configuration.
5. Test the cron manually with a small batch size.
6. Check the module logs and the server/PHP error log.
7. Make sure you are using the latest released version from GitHub.

## Useful information to include

When requesting support, include:

- PrestaShop version,
- PHP version,
- module version,
- database type and version if known,
- the exact error message,
- whether the issue happens in CLI cron, HTTP cron or the module configuration,
- relevant log output without passwords, tokens or customer data.
